# PPT Generator

将讲稿转换为极简科技感 HTML 演示稿。论文汇报默认使用 16:9 宽屏布局；主题支持亮色与暗色两种风格。

## 设计哲学

- **极简主义** - 一屏只讲一件事
- **强视觉对比** - 深色背景 + 白色文字
- **高留白** - 禁止密集排版
- **强节奏感** - 让观众想继续看

## 生成流程（必须严格遵循）

### Step 1: 读取讲稿
读取用户原始讲稿，不修改原稿内容。

### Step 2: 生成提炼版讲稿
将内容精简、增强冲击力、适配演示场景，输出 Markdown 格式。

### Step 3: 生成乔布斯风标题
为每个章节生成标题，必须满足：
- ≤12 字
- 采用以下形式之一：对比式、问题式、断言式、数字式、比喻式
- 自检：是否让人想继续听？

### Step 4: 设计幻灯片结构
规划页面顺序和类型，参考 [references/slide-types.md](references/slide-types.md)。

### Step 5: 生成HTML
使用 [assets/template.html](assets/template.html) 作为基础模板生成完整HTML。

### Step 6: 填充内容
添加动态背景光斑、交互逻辑、平滑切换动画。

## 输出顺序（必须依次输出）

1. **提炼后的讲稿**（Markdown）
2. **幻灯片结构大纲**
3. **完整HTML代码**

## 布局与主题默认规则

<!-- Why: 论文汇报更常在组会、教室或会议室投屏展示，竖屏默认不适合此类阅读场景；同时不同场景需要明暗两套主题。 Scope: 调整 skill 的默认布局与主题策略，使论文汇报默认生成宽屏，并为后续 HTML 生成提供亮色/暗色双主题约束。 Verify: 后续使用该 skill 生成论文汇报时，检查默认大纲为论文结构、默认比例为 16:9 宽屏，且模板可在亮色与暗色主题间切换。 -->

- 当任务是**论文汇报、paper reading、组会分享、文献解读**时：
  - 默认使用 **16:9 宽屏布局**
  - 默认主题使用 **暗色学术主题**
  - 若用户未指定，优先输出适合投屏的宽屏 HTML，而不是竖屏页面
- 当用户明确要求移动端展示、短视频风格或竖屏页面时，才切回 **9:16 竖屏**
- 主题必须支持两套：
  - **暗色主题**：深色背景、高对比、科技感强，适合组会与投屏
  - **亮色主题**：浅色背景、论文感更强，适合教室汇报、答辩或打印截图

## 论文汇报默认结构

<!-- Why: 论文阅读、组会汇报和文献分享需要稳定的学术叙事主线，避免内容顺序漂移。 Scope: 当任务是论文/paper reading/组会汇报/文献分享时，优先按固定章节结构组织幻灯片。 Verify: 后续使用该 skill 生成论文汇报时，检查结构大纲默认包含“研究背景→研究方法→实验结果→分析讨论→启发→Q&A”。 -->

- 当任务是**论文汇报、paper reading、组会分享、文献解读**时，封面之后默认按照以下一级结构组织：
  1. **研究背景**
  2. **研究方法**
  3. **实验结果**
  4. **分析讨论**
  5. **启发**
  6. **Q&A**
- 如果论文内部包含多个方法或多个实验模块，可以在上述一级结构下继续拆成多页，但**不要跳出这条主线**。
- 若用户未明确指定顺序，默认优先采用这套论文汇报结构，而不是自由散开的叙事方式。

## 论文封面页要求

<!-- Why: 论文汇报首页需要快速交代论文身份信息，方便听众在最短时间建立出处与作者背景。 Scope: 约束论文类演示的第一页内容字段与顺序，要求生成器默认输出完整的论文元信息。 Verify: 后续生成论文汇报时，检查第一页默认包含标题、发表时间、会议或期刊、作者、第一作者机构。 -->

- 当任务是**论文汇报、paper reading、组会分享、文献解读**时，第一页默认必须包含以下字段，并按该顺序组织：
  1. **论文名字（title）**
  2. **换行**
  3. **论文发表时间**
  4. **论文发表在哪个期刊或者会议**
  5. **换行**
  6. **作者名字**
  7. **第一作者的机构**
- 若用户没有提供完整元信息：
  - 优先从原论文首页、PDF 元数据或用户给出的材料中提取
  - 缺失项要显式留占位，不要默默省略
- 封面页中的论文元信息应保持简洁，不要挤成大段正文；推荐做成标题下的 2~3 行元信息区。

## 视觉规范速查

| 项目 | 规范 |

| 论文汇报默认比例 | 16:9 宽屏 |

| 暗色背景 | #000000 / #0a0a0a + 模糊光斑动画 |
| 亮色背景 | #f7f9fc / #eef3ff + 柔和高光 |

| 英文字体 | Inter / Roboto |
| 标题字重 | font-black / font-bold |
| 正文字重 | font-light / font-normal |

## 交互要求

- 键盘 ← → 翻页
- 底部进度导航条
- 平滑切换动画

## 技术栈

- TailwindCSS（国内CDN）
- 复杂页面使用 Vue3（CDN）
- 单个HTML文件，可直接打开运行

## 严禁行为

- 堆字 / 密集排版
- 花哨配色
- 复杂图表
- 偏离极简科技风
- 在论文汇报场景下默认输出竖屏页面
- 只支持单一暗色主题，不提供亮色替代方案

## 默认规则

- 未指定页数：自动生成 8~20 页
- 未指定风格：默认极简科技风
- 未指定布局：论文汇报默认 16:9 宽屏，其他场景按内容判断
- 未指定主题：默认暗色主题，同时保留亮色主题可切换实现
