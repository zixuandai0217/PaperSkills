XSKILL

Continual Learning from Experience and Skills in Multimodal Agents

发表时间：March 16, 2026
发表平台：arXiv Preprint
作者：Guanyu Jiang · Zhaochen Su · Xiaoye Qu · Yi R. (May) Fung
第一作者机构：Hong Kong University of Science and Technology / Zhejiang University

一篇关于多模态 agent 如何不改参数，却能持续从轨迹中学会更好用工具的论文。

研究背景

为什么值得讲

多模态 agent 已能用工具，但依然缺少稳定的跨任务成长机制。

研究方法

怎么做

用 experience + skill 双流知识，把轨迹经验变成可检索、可适配的外部能力。

实验结果

最关键结果

Average@4 相对 tool-only 提升 +2.58~6.71，并支持跨模型迁移。

01 / 研究背景 Problem & Motivation
多模态 agent 已能调工具，但仍不够会“做事”

问题不只是有无工具
而是不会持续变强

Pain Point A

工具使用低效

简单题走太多步，复杂题却又探索不够深；试错成本高，turn budget 被浪费。

Pain Point B

工具编排僵硬

多数系统偏单路径执行，难以根据任务场景灵活切换工具与工作流。

  • 作者视角：关键不是继续训练参数，而是让 agent 从过往轨迹里沉淀外部知识。
  • 多模态难点：很多决策线索来自视觉观察，仅看文本轨迹不足以召回正确经验。
Figure 1 与引言共同强调：视觉语义错位会导致旋转、裁剪、检索等关键步骤缺失。
01 / 研究背景 Core Idea
一句话：既记经验，也存技能

让 agent 同时拥有
experience + skill

experience

动作级、战术性

回答“在什么条件下，应该做什么动作”。适合工具选择、错误恢复、局部决策。

  • 结构：condition → action
  • 形式：短文本、可检索、上下文敏感
  • 价值：让 agent 更会临场变招
skill

任务级、结构化

回答“这类问题通常怎么规划”。适合工作流、工具模板、可复用步骤。

  • 结构：workflow + tool templates
  • 形式：Markdown skill document
  • 价值：让 agent 更稳、更少犯结构性错误
[FACT]

作者的核心判断：skill 提升工具使用鲁棒性，experience 提升工具编排灵活性；两者互补，而不是替代关系。

02 / 研究方法 Overall Framework
视觉 grounding 是贯穿始终的主线

不是静态记忆库
而是一个持续回流的闭环

Phase I

Accumulation

从多路径 rollout 中提炼 Experience Bank 与 Skill Library。

↻
Phase II

Solving

按子任务检索，再结合当前图像把经验与技能适配进 prompt。

两个模型分工

MLLMexec 负责执行工具推理，MLLMkb 负责提炼、整合、改写知识。

知识回流

使用历史会被送回积累阶段，形成 continual learning loop。

[FACT]

论文当前实验是 single accumulation-then-test cycle，但整体架构被设计成可继续迭代 refined knowledge。

02 / 研究方法 Knowledge Accumulation
目标：把轨迹变成可复用知识

Phase I 重点不在“存日志”
而在抽象与整合

A. Rollout Summary

多次 rollout 后做视觉 grounding 的摘要：保留关键决策点、失败原因、触发动作的视觉证据。

↓
B. Cross-Rollout Critique

比较成功与失败轨迹，提炼可泛化的 condition-action 经验，避免只记“发生了什么”。

↓
C. Knowledge Consolidation

按相似度、长度与质量做 merge / refine / delete，把零散片段变成干净的 knowledge base。

关键词：visual evidence、hierarchical consolidation、去冗余、跨 rollout 归因。
02 / 研究方法 Inference & Adaptation
目标：把通用知识改写成当前任务可执行提示

Phase II 的关键是
检索之后还要适配

1. Task Decomposition Retrieval

先把问题拆成多个子任务，再分别检索 top-k experiences，避免原始 query 一次检索漏掉关键维度。

2. Experience Rewrite

把泛化经验重写到当前图像和问题上：保留结构，但丢掉不相关细节，变成当前 task 的 actionable hint。

3. Skill Adaptation & Injection

裁剪 skill document 的无关段落，把改写后的经验嵌入 workflow，再以 non-prescriptive reference 注入 prompt。

[FACT]

作者强调：不是把技能模板硬塞给 agent，而是给出可参考、可偏离的结构化提示，让 agent 保留 improvise 的空间。

04 / 分析讨论 Ablation & Discussion
两类知识都重要，但积累质量更重要

消融给出的信息非常清楚：
Phase I 比 Phase II 更关键

What is necessary
  • w/o Experience：Average@4 下降 -3.04
  • w/o Skill：Average@4 下降 -3.85
  • w/o Experience Manager：-4.09
  • w/o Skill Manager：-3.62
What changes behavior
  • VisualToolBench 上 code interpreter 使用率：66.63% → 74.49% → 76.97%
  • MMSearch-Plus 上 code interpreter：6.18% → 13.21%
  • image search：15.43% → 24.63%
  • 说明 experience 主要影响战术工具选择。
[FACT]

作者据此认为：先把知识积累好 比“检索写得多聪明”更关键；skill 更像减少结构性错误，experience 更像引导灵活工具编排。

05 / 启发 Takeaways
这篇文章真正回答的是“为什么要双流知识”

对我来说，XSKILL 的价值
在于把“记忆”做成了可执行系统

[FACT]
  • skill 与 experience 是互补的，不是二选一。
  • 视觉 grounding 对多模态 agent 至关重要。
  • 知识可以跨模型迁移，说明它不只是模型专属技巧。
  • 论文强调 superior zero-shot generalization。
[PLAN]
  • 当前验证仍主要是单轮 accumulation-then-test。
  • 真正的 lifelong refinement 还需要更长周期实验。
  • 知识质量依赖 rollout 多样性与 critique 质量。
  • 后续可关注成本、时延和更复杂工具生态。

一句 takeaway：如果说传统 agent 的问题是“只会做当前题”，那 XSKILL 在尝试让它把一次次做题经验，真正转化成下次能用的外部能力。
